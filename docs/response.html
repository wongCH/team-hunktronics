Outcome

What changed

The implementation follows the scale-first plan without adding a database or another runtime dependency.

Identity

SOUL.md is canonical

Every identity is stored at agents/<id>/SOUL.md. agents.json contains metadata, soulPath, and a compact routing manifest only.

Routing

Specialist first, bounded always

Managers score compact manifests, choose the best specialist branch, then fall back to the team lead and finally themselves. Full souls load only after selection.

Runtime

Admission replaces bursts

Provider calls respect global, connection, team, and agent limits. Schedules start four at a time and retry with distinct idempotency keys.

Memory

The learning loop closes

Runs retain bounded outcomes and artifact references. Same-day appends serialize, retrieval is scope-limited, and compression proposals survive restart.

Lifecycle

Removal is recoverable

Active runs block removal. Archival stops routing but retains identity and dependent records; the UI exposes explicit restoration.

Renderer

Large maps show one branch

Above 60 agents, the map renders the root/lead spine plus the selected branch. Streaming activity no longer reruns Dagre topology layout.

Durability

Memory and lifecycle

Run completion

  • Capture at most 1,000 characters of outcome.
  • Record artifact references, status, and errors.
  • Do not duplicate the raw private prompt.

Serialized learning

  • Queue by agent and date.
  • Preserve all concurrent terminal entries.
  • Persist compression proposals for restart recovery.

Retention

  • History retains for 30 days.
  • Daily logs archive after 60 days.
  • Archives retain for 365 days.

Decision

Do not add SQLite yet

Agent metadata is low-churn, atomically written, and cached. Hot paths now have serialization and admission control. A database today would add migration and packaging cost without fixing a measured bottleneck.

Revisit when queued runs must survive restart, cross-record queries dominate, or measured JSON contention exceeds the current contract.

Known boundaries

Intentionally out of scope

No execution workspace

The agent folder owns identity, not a checkout, shell sandbox, or working directory.

Admission is process-local

Queued provider calls do not survive restart. Durable queues trigger the SQLite phase.

Retrieval stays lexical

Add embeddings only after evaluation demonstrates material lexical misses.

Schedules require the app

Backpressure and retries work while Electron is open; there is no OS daemon.

One local workspace

Target roughly 100–150 agents per workspace, then federate for ownership and scale.

No embedding provider

Memory remains local unless a future user explicitly configures an external provider.
